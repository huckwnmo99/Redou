# Selected Design Direction

선택된 기준 디자인:

- `design-options-v3/21_reference_collector.html`

## 왜 이 방향을 선택했는가

이 방향은 현재까지 만든 시안 중 아래 특성이 가장 잘 맞는다.

- 구조가 단순하고 직관적이다.
- 카드형 라이브러리와 좌측 카테고리/폴더 구조가 바로 이해된다.
- 너무 무겁거나 과장되지 않고, 실제 앱으로 만들었을 때 안정적이다.
- 참고 이미지와 가까운 정돈된 도구형 UI 느낌이 있다.
- 폴더 기반 분류, 카드 기반 논문 탐색, 빠른 전환 구조와 잘 맞는다.

## 실제 앱으로 가져갈 핵심 요소

### 1. 전체 톤

- 깔끔한 데스크톱 도구 느낌
- 과한 장식보다 정리된 패널 구조 우선
- 기본은 밝은 배경과 얇은 경계선
- glassmorphism은 약하게만 사용하거나, 필요하면 거의 제거 가능

### 2. 레이아웃

- 좌측: 폴더 / 카테고리 / 상태 필터
- 중앙: 논문 카드 그리드 또는 리스트
- 상세 진입 시: paper detail / PDF / notes / figures로 전환
- 필요 시 우측 inspector panel 추가

### 3. 핵심 UI 패턴

- 카드 기반 paper browsing
- 폴더 트리 또는 카테고리 리스트
- 선택된 항목이 명확히 보이는 active state
- 과한 그림자 대신 얕은 elevation 사용
- 검색창은 상단 고정형이 잘 맞음

### 4. 시각 원칙

- 색은 구조 보조 정도로만 사용
- 강조는 선택 상태, 경계선, 배경 농도 차이로 표현
- 하이라이트 색은 논문 annotation 의미 전달에 우선 사용
- 앱 자체 UI 색이 annotation 색을 방해하면 안 됨

## 이 방향으로 다음에 해야 할 일

1. `21` 스타일 기준으로 실제 app shell 화면 다시 설계
2. Library 화면, Paper Detail 화면, PDF Reader 화면을 같은 디자인 언어로 통일
3. 폴더 트리 + 논문 카드 + 검색창 + 상태 배지를 실제 컴포넌트 구조로 내리기
4. detached panel 기능도 이 디자인 언어 안에서 자연스럽게 다시 해석하기

## 한 줄 결론
