# Redou — AI 연구 논문 분석 플랫폼

## 아이템명

**Redou** (Research Document Understanding)

## 아이템 개요 (500자 이내)

Redou는 연구자가 수십~수백 편의 논문을 로컬에서 관리하고, AI가 논문 데이터를 자동으로 구조화하여 비교 분석 테이블과 질의응답을 제공하는 데스크탑 앱이다. PDF를 임포트하면 텍스트, 테이블, 수식, 그림을 자동 추출하고, 시맨틱 검색 인프라(임베딩 + BM25 하이브리드 검색)를 구축한다. 핵심 기능은 "비교 테이블 자동 생성"으로, 사용자가 "이 논문들의 흡착량을 비교해줘"라고 요청하면 AI가 각 논문에서 수치를 독립 추출하고 병합하여 정확한 비교표를 생성한다. 기존 논문 관리 도구(Mendeley, Zotero)는 메타데이터 관리에 그치고, ChatGPT 등 범용 AI는 논문 내부 수치를 정확히 처리하지 못한다. Redou는 논문 내부 데이터까지 구조화하여 "읽기"를 넘어 "분석"을 자동화한다. 모든 데이터는 로컬에 저장되어 기업/연구기관의 보안 요구를 충족한다.

## 핵심 기능 구성

### 1. PDF 자동 처리 파이프라인 (구현 완료)

| 기능 | 설명 |
|------|------|
| PDF 텍스트 추출 | MinerU 구조화 파싱 + pdfjs 폴백 |
| 메타데이터 추출 | GROBID: 제목, 저자, DOI, 참고문헌 자동 인식 |
| Figure/Table/Equation 감지 | 휴리스틱 감지 + OCR(GLM-OCR) 보강 |
| 테이블 OCR | 논문 내 테이블을 HTML 구조화 데이터로 변환 |
| 수식 OCR | UniMERNet: 수식 이미지 → LaTeX 변환 |
| 시맨틱 임베딩 | 청크 단위 벡터 임베딩 (2048-dim, pgvector) |

### 2. AI 비교 테이블 생성 (핵심 차별점, 구현 완료 + 고도화 중)

| 기능 | 설명 | 상태 |
|------|------|------|
| 의도 분석 | 사용자 요청에서 비교 항목/열 자동 도출 | 구현 완료 |
| Hybrid Search | BM25 키워드 + 벡터 의미 검색 결합 (RRF 퓨전) | 구현 완료 |
| Cross-encoder Reranker | 검색 결과 재정렬로 노이즈 제거 | 구현 완료 |
| Contextual Chunking | 청크에 논문/섹션 맥락 반영 (정확도 18% 향상) | 구현 완료 |
| 테이블 우선 검색 | 논문 원본 테이블 데이터를 우선 활용 | 구현 완료 |
| SRAG 2단계 추출 | 논문별 독립 추출 → 병합 (수치 혼동 방지) | 구현 예정 |
| NULL 셀 자동 재검색 | 누락 데이터 감지 → 타겟 재검색 | 계획됨 |
| CRAG 셀 검증 | 생성된 수치를 원문과 대조 검증 | 계획됨 |
| Guardian 검증 | AI 안전성 검증 (hallucination 감지) | 구현 완료 |

### 3. 논문 Q&A (구현 완료)

| 기능 | 설명 |
|------|------|
| 자연어 질의응답 | "이 논문의 핵심 결론은?" → 출처 기반 답변 |
| 출처 귀속 | 답변에 논문명 + 섹션 참조 자동 포함 |
| Table/Q&A 모드 분리 | 용도별 최적화된 검색 전략 자동 적용 |

### 4. 논문 관리 & 리더 (구현 완료)

| 기능 | 설명 |
|------|------|
| PDF 리더 | 내장 뷰어 + 하이라이트 + 주석 |
| 폴더 관리 | 논문을 폴더별로 정리 |
| 시맨틱 검색 | 키워드가 아닌 의미 기반 논문 검색 |
| Figure/Table 갤러리 | 논문 내 그림/테이블/수식 브라우징 |
| 노트 워크스페이스 | 논문별 연구 노트 작성 |

### 5. 지식 그래프 & 관계 추론 (계획됨)

| 기능 | 설명 | 상태 |
|------|------|------|
| Ontology 기반 엔티티 추출 | 논문에서 물질, 공정 조건, 성능 지표 등 도메인 엔티티를 자동 추출 | 계획됨 |
| 관계 그래프 구축 (GraphRAG) | 엔티티 간 인과관계 매핑 (예: "압력 → 순도 → 회수율") | 계획됨 |
| 인용 네트워크 탐색 | 참고문헌 체인으로 관련 논문 발견 + 연구 계보 시각화 | 계획됨 |
| 그래프 + 벡터 융합 검색 | 의미 검색과 관계 추론을 결합한 멀티모달 검색 | 계획됨 |
| 지식 그래프 시각화 | 논문 간 관계를 인터랙티브 그래프로 탐색 | 계획됨 |
| 멀티홉 추론 | "A 물질이 B 조건에서 효과적이라면, C 공정에 적용 가능한가?" 같은 복합 질문 | 계획됨 |

**Ontology 설계 방향:**
```
[물질/소재] --흡착--> [가스/분자]
[물질/소재] --사용조건--> [온도, 압력, 유속]
[실험] --측정--> [성능지표: 흡착량, 순도, 회수율, 선택도]
[논문] --보고--> [실험]
[논문] --인용--> [논문]
```

도메인별 Ontology 템플릿을 제공하여, 화학공학뿐 아니라 바이오, 재료, 에너지 등 다른 연구 분야에도 확장 가능.

## 경쟁 우위

| 비교 대상 | 한계 | Redou 차별점 |
|-----------|------|-------------|
| Mendeley / Zotero | 메타데이터 관리만, 논문 내부 데이터 분석 불가 | 테이블/수치/수식까지 구조화 |
| ChatGPT / Claude | 업로드 논문 수 제한, 수치 hallucination | 로컬 RAG로 정확한 수치 추출 |
| Elicit / Semantic Scholar | 초록 수준 분석, 본문 수치 비교 불가 | 본문 테이블까지 파싱하여 비교표 생성 |
| Connected Papers | 인용 관계만, 내용 분석 없음 | 내용 기반 의미 검색 + 데이터 추출 |
| Notion AI / Obsidian | 범용 노트, 논문 특화 기능 없음 | 논문 구조 이해 + 도메인 Ontology |

**핵심 차별점**: 논문 내부의 수치 데이터를 AI가 정확하게 추출하여 비교 분석표를 자동 생성하고, Ontology 기반 지식 그래프로 논문 간 관계를 추론하는 유일한 로컬 데스크탑 솔루션.

## 타겟 사용자

1. **대학원생 / 연구원**: 문헌 조사 시 수십 편의 논문에서 실험 조건·결과를 비교표로 정리하는 반복 작업 자동화
2. **기업 R&D**: 특허/논문에서 경쟁사 기술 데이터를 수집·비교 (보안 요구 → 로컬 처리)
3. **리뷰 논문 저자**: 대량의 선행 연구를 체계적으로 정리하여 비교 테이블 작성
4. **연구 그룹 리더**: 팀원들의 문헌 조사 결과를 통합하고 연구 방향 설정

## 추가 제안 기능 (Claude 제안)

> 아래는 현재 로드맵에 없지만, 제품 경쟁력과 사업성 강화를 위해 제안하는 기능입니다.

### A. 비교 테이블 내보내기 (Export)

| 형식 | 용도 |
|------|------|
| Excel (.xlsx) | 연구자가 직접 가공/편집 |
| LaTeX (.tex) | 논문 작성 시 바로 삽입 |
| CSV | 데이터 분석 도구(Python, R)에 입력 |
| Word (.docx) | 보고서/제안서에 삽입 |
| Clipboard (HTML) | 다른 앱에 붙여넣기 |

**제안 이유**: 비교 테이블을 생성하는 것까지는 좋지만, 실제 연구자는 그 테이블을 논문이나 보고서에 넣어야 한다. 내보내기 없으면 스크린샷을 찍게 되고, 그러면 "AI가 만들어줬다"의 가치가 반감된다.

### B. 연구 트렌드 시계열 분석

논문 라이브러리에서 시간축(발행 연도)에 따른 데이터 변화를 자동 시각화:
- "CO2 흡착량이 2015년부터 2025년까지 어떻게 변했나?"
- 물질별/공정별 성능 추이를 차트로 자동 생성
- 연구 방향 전환점 감지 ("2020년 이후 MOF 기반 연구가 급증")

**제안 이유**: 비교 테이블이 "현재 스냅샷"이라면, 트렌드 분석은 "시간에 따른 변화"를 보여준다. 리뷰 논문 작성자와 R&D 전략 수립자에게 매우 유용한 기능이며, 경쟁 제품에 없는 차별점이 된다.

### C. 자동 문헌 조사 보고서 생성

사용자가 연구 주제를 입력하면, 라이브러리 내 관련 논문을 종합하여 구조화된 문헌 조사 보고서를 자동 생성:
- 연구 배경 요약
- 주요 접근법 분류
- 비교 테이블 (자동 생성)
- 미해결 문제 / 향후 연구 방향
- 참고문헌 목록

**제안 이유**: 문헌 조사 보고서 작성은 대학원생이 가장 많은 시간을 쓰는 작업 중 하나. 비교 테이블은 이 보고서의 일부분이므로, 보고서 전체를 생성하면 단위 기능에서 "워크플로우 자동화"로 발전한다.

### D. 도메인별 템플릿 마켓

분야별로 자주 쓰이는 비교 항목과 Ontology를 템플릿으로 제공:
- **화학공학**: 흡착량, 선택도, 재생 온도, BET 면적, 기공 크기...
- **바이오/제약**: IC50, 바이오어밸리어빌리티, 투여 경로, 부작용...
- **재료과학**: 인장강도, 탄성계수, 밀도, 열전도도...
- **에너지**: 효율, 출력 밀도, 수명 사이클, 비용...

사용자가 "흡착 공정 비교" 템플릿을 선택하면, 비교 열과 Ontology가 자동 설정되어 바로 테이블 생성 가능.

**제안 이유**: 현재는 사용자가 "어떤 열을 비교할지"를 매번 자연어로 설명해야 한다. 도메인 템플릿이 있으면 클릭 한 번으로 전문적인 비교 테이블을 생성할 수 있고, 이 템플릿 자체가 커뮤니티/마켓플레이스 비즈니스 모델이 될 수 있다.

### E. 협업 & 클라우드 동기화 (Pro 버전)

| 기능 | 설명 |
|------|------|
| 팀 라이브러리 공유 | 연구 그룹이 논문 + 분석 결과를 공유 |
| 권한 관리 | 읽기/편집/관리자 역할 분리 |
| 클라우드 동기화 | 로컬 ↔ 클라우드 선택적 동기화 |
| 공동 노트 편집 | 같은 논문에 여러 연구자가 주석/노트 |
| 활동 로그 | 누가 어떤 논문을 분석했는지 추적 |

**제안 이유**: 로컬 전용은 보안이 강점이지만, 연구 그룹 단위 사용에서는 협업이 필수. "로컬 기본 + 클라우드 옵션" 모델로 개인(무료/저가) → 팀(유료) 수익 구조를 만들 수 있다.

### F. 논문 자동 수집 (Watch & Import)

| 기능 | 설명 |
|------|------|
| 키워드/저자 Watch | 새 논문이 나오면 자동 알림 + 임포트 |
| DOI/URL 일괄 임포트 | 참고문헌 목록에서 DOI 추출 → 일괄 다운로드 |
| 브라우저 확장 | 웹에서 논문 보다가 원클릭 임포트 |
| arXiv/PubMed 연동 | 데이터베이스에서 직접 검색 + 임포트 |

**제안 이유**: 현재는 사용자가 PDF를 수동으로 임포트해야 한다. 논문 발견 → 임포트 → 분석의 전체 워크플로우를 자동화하면, "분석 도구"에서 "연구 어시스턴트"로 포지셔닝이 올라간다.
