# SRAG 스타일 2단계 추출 (추출 → 조립 분리)

> 상태: 💡 아이디어 | 등록일: 2026-04-07 | 출처: [Table RAG 개선 제안서](../01-Idea/Table_RAG_improvement_report.md#5-srag-스타일-2단계-추출-추출--조립-분리)

## 배경
현재 RAG 결과 전체를 LLM에 한 번에 전달하여 테이블 생성. 다수 논문의 데이터가 섞이면 LLM이 수치를 혼동하거나 누락.

## 핵심 아이디어
- SRAG (2025, arXiv:2503.01346): 비정형 텍스트 → 관계형 테이블 변환 → SQL 추론
- Step 1: 논문별 독립 LLM 호출로 구조화 데이터(JSON) 추출
- Step 2: 전체 JSON 병합 → NULL 필드 감지 → 비교 테이블 조립
- NULL 필드가 Agentic 재검색의 트리거가 됨

## 예시
```
논문A 청크 → LLM "이 논문에서 {온도, 압력, 순도} 추출해" → {온도: 300K, 압력: 1atm, 순도: 99.5%}
논문B 청크 → LLM "이 논문에서 {온도, 압력, 순도} 추출해" → {온도: 350K, 압력: NULL, 순도: 98.1%}
→ 논문B 압력 NULL → 재검색 트리거 가능
```

## 예상 영향
- Electron: `llm-orchestrator.mjs` Table Agent 파이프라인 대폭 수정
- LLM 호출: 논문 수 × 1회 (기존 1회 → N회)
- 정확도 대폭 향상 기대

## 선행 조건
- Hybrid Search + Reranker가 선행되면 각 논문별 추출 정확도 향상

## 주의사항
- LLM 호출 횟수 증가 — 논문 수에 비례
- 구조화 JSON 스키마 설계 필요
