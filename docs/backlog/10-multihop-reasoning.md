# 쿼리 분해 + 멀티홉 추론

> 상태: 💡 아이디어 | 등록일: 2026-04-06 | 출처: [RAG 설계 제안서](../01-Idea/Rag_design_report.md#idea5)

## 배경
"Zeolite 5A PSA 7단계가 5단계보다 효율적인 이유?"처럼 복잡한 질문은 1회 검색으로 답할 수 없음.

## 핵심 아이디어
- 복잡한 질문 → 서브 질문으로 분해 → 각각 검색 → 결과 통합
- 질문 복잡도에 따라 전략 분기: 단순(1회) / 비교(병렬) / 인과(멀티홉)

## Redou 기존 자산
- `llm-orchestrator.mjs`가 이미 의도 분석 + 검색 쿼리 생성 수행
- 확장하여 멀티홉 루프 추가 가능

## 예상 영향
- Electron: `llm-orchestrator.mjs` 확장 또는 새 모듈
- 응답 시간: 서브 질문 수에 비례하여 증가

## 선행 조건
- CRAG, Hybrid Search가 먼저 구축되어야 효과적
