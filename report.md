# Redou V1 - 미해결 사항 보고서

## 1. 미구현 기능

| 항목 | 상태 | 설명 |
|------|------|------|
| OCR 추출 | 구현됨 (조건부) | 전체 파이프라인 구현 완료 (래스터화→Tesseract→텍스트). 단, tesseract+pdftoppm/magick 시스템 설치 필요. 없으면 자동 스킵 |
| 저자 데이터 | 미구현 | `paper_authors` 테이블 없음. 프론트엔드 authors 항상 `[]` |
| 인용수 | 미구현 | 저장 컬럼/테이블 없음. 항상 `0` |
| RLS 정책 | 미구현 | 모든 테이블에 Row-Level Security 없음 |
| Google OAuth | 부분 구현 | UI·설정 존재하나 로컬 Supabase에서 실제 인증 플로우 미검증 |
| 레거시 렌더러 제거 | 미완료 | `apps/desktop/src` 아직 존재 |

## 2. 데이터 레이어 버그

| ID | 심각도 | 내용 |
|----|--------|------|
| A-1 | MEDIUM | NoteKind 매핑 손실 — DB→프론트 변환 시 여러 타입이 하나로 합쳐짐, 라운드트립 불가 |
| A-2 | MEDIUM | `fetchPaperSignals()` — notes/figures/jobs 전체 테이블 풀스캔. 논문 많아지면 성능 병목 |
| A-8 | MEDIUM | 논문이 여러 폴더에 속할 수 있지만 첫 번째 폴더만 표시 |
| A-9 | MEDIUM | 하이라이트 중복검사 anchorId가 페이지 단위 → 같은 페이지 다른 텍스트 구분 불완전 |
| A-10 | MEDIUM | 활성 프리셋 없으면 하이라이트/노트 생성 시 크래시 |

## 3. Electron/IPC 버그

| ID | 심각도 | 내용 |
|----|--------|------|
| D-9 | **HIGH** | 추출 저장 트랜잭션 없음 — 삭제 후 삽입 실패 시 데이터 유실 |
| D-5 | MEDIUM | 프로세싱 워커 레이스 컨디션 (boolean 가드) |
| D-4 | MEDIUM | 백업 복원 시 기존 데이터 미삭제 → 잔여 데이터 남음 |
| D-10 | MEDIUM | 백업 복원 경로 검증 없음 → 임의 파일 읽기 가능 |
| D-11 | LOW | 백업 복원 upsert 에러 무시 |

## 4. 프론트엔드 버그

| ID | 심각도 | 내용 |
|----|--------|------|
| E-4 | MEDIUM | 줌 레벨에 따라 하이라이트 오버레이 미세 어긋남 |
| F-4 | LOW | 하이라이트 하나 수정 중이면 모든 하이라이트 컨트롤 비활성화 |
| F-2 | LOW | 섹션/하이라이트 최대 6개만 표시, "더보기" 없음 |
| G-3 | LOW | 폴더 순환 참조 방지 없음 → 무한재귀 가능 |

## 5. 우선순위 제안

1. **D-9** (추출 트랜잭션) — 데이터 유실 위험, 즉시 수정 필요
2. **A-10** (프리셋 없을 때 크래시) — 사용자가 바로 마주침
3. **A-2** (풀스캔 성능) — 논문 수 늘어나면 체감 성능 저하
4. **OCR 통합** — 스캔 PDF 지원의 핵심
5. **레거시 렌더러 정리** — 코드베이스 단순화
